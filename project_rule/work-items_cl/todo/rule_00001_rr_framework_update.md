## 배경

사용자의 R&R(Roles & Responsibilities) 프레임워크를 프로젝트 워크플로우에 매핑하여,
`workflow_rules.md`와 `CLAUDE.md`(최상위 + 하위 프로젝트)를 업데이트한다.

### R&R 정의 (사용자 원안)

| 역할 | 정(Primary) | 부(Secondary) | 비고 |
|------|-------------|---------------|------|
| **평가자** | 사용자 | — | 최종 승인/검증 |
| **커맨더** | 사용자(요청), Claude(설계) | Codex | Todo 문서 작성, 전략 설계 |
| **엑터** | Codex | Claude | 구현은 Codex, 검증/보조는 Claude |

### 프롬프트 핵심 개념과 프로젝트 매핑

| 프롬프트 개념 | 프로젝트 매핑 | 설명 |
|---|---|---|
| 글로벌 맵 (상위 좌표계) | `workflow_rules.md` (Base Rules) | 변하지 않는 상위 기준 좌표 |
| Real Self (방향 감각) | 프로젝트 비전/목표 (사용자 정의) | 의식 이전의 방향성 센서 |
| Meta Self-ego (운영체계) | `CLAUDE.md` + Todo 문서 체계 | 방향을 전략/기준으로 번역 |
| Ego (Actor) | 실제 구현 (Codex 정, Claude 부) | 현장에서 행동하는 실행자 |
| 롤모델 Ego (미래 실행 패턴) | Done 이력 + 검증된 패턴 | 과거 성공 경험의 축적 |
| 인지조정자 | Claude (커맨더 역할) | 좌표 변환기, 구조 해석 |
| 정렬 루프 | Todo → Issue → Done 사이클 | 감지 → 번역 → 실행 → 해석 → 재번역 |

## 3축 평가 기준

### 1. 자동화 정도 (사용자에게 물어보지 않고 수행 가능한 범위)

| 영역 | 자동화 수준 | 비고 |
|------|------------|------|
| Todo 문서 구조 생성 | **높음** | 템플릿 기반 자동 생성 가능 |
| Kickoff 분석 (rule101) | **높음** | 기존 코드/문서 기반 범위 분석 |
| 이슈 문서 생성/갱신 | **높음** | 작업 중 발견 사항 자동 기록 |
| 코드 구현 | **중간** | Codex가 주로 수행, 명확한 스펙 필요 |
| Done 판정 (rule201) | **높음** | C-1~C-4 기준 자동 판정 가능 |
| 아키텍처 결정 | **낮음** | 사용자 승인 필수 |
| 크로스 프로젝트 영향 분석 | **중간** | Impact Matrix 자동 참조, 판단은 사용자 |
| 검증/테스트 수행 | **중간** | 테스트 실행은 자동, 합격 기준은 사용자 |

### 2. 능력 매핑 (Claude vs Codex 적합성)

| 업무 | Claude 적합도 | Codex 적합도 | 권장 배정 |
|------|-------------|-------------|-----------|
| 작업 범위 분석/설계 | ★★★ | ★★ | Claude 정 |
| Todo/Issue 문서 작성 | ★★★ | ★★ | Claude 정 |
| 코드 구현 | ★★ | ★★★ | Codex 정 |
| 코드 리뷰/검증 | ★★★ | ★ | Claude 정 |
| 테스트 작성/실행 | ★★ | ★★★ | Codex 정 |
| CPCR 작성 | ★★★ | ★ | Claude 정 |
| 규칙 해석/적용 | ★★★ | ★ | Claude 정 |
| 리팩토링 | ★★ | ★★★ | Codex 정 |
| 현황 대시보드 (rule200) | ★★★ | ★ | Claude 정 |
| Done 판정 (rule201) | ★★★ | ★ | Claude 정 |

### 3. 목표 설정 가능 정도 (Self 평가 역할)

| 항목 | 가능 여부 | 설명 |
|------|----------|------|
| 작업 완료 여부 판정 | **가능** | C-1~C-4 기준 객관적 체크 |
| 코드 품질 평가 | **부분 가능** | 정적 분석 + 패턴 매칭은 가능, 비즈니스 적합성은 사용자 판단 |
| 규칙 준수 여부 체크 | **가능** | workflow_rules.md 기준 자동 체크 |
| 크로스 프로젝트 영향 감지 | **가능** | Impact Matrix 자동 참조 |
| 전략-실행 간 갭 분석 | **가능** | Todo vs 실제 구현 diff 비교 |
| 최종 승인 | **불가** | 평가자(사용자) 전용 |
